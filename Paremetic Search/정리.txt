이진 탐색 -> 탐색 범위를 절반씩 좁혀가며 데이터를 탐색, 시작점, 끝점, 중간점을 뜻하는 변수 3개의 값을 변화시키면서 탐색

매개 변수 탐색 -> 최적화 문제를 결정 문제로 풀 수 있는 기술
최적화 문제 : 어떤 알고리즘의 최적 솔루션을 찾아내는 것
결정 문제 : 답이 이미 결정되어있다고 보고 문제를 푸는 것 => 징검다리 문제? 


-> 매개 변수 탐색 문제를 언제 사용할 수 있을까?
1. 결정 문제로 풀 수 있는 문제
2. 어떤 시점까지는 조건을 만족하지만, 그 시점 이후로는 조건을 만족하지 않은 경우에 최대값 찾기
3. 어떤 시점까지는 조건을 만족하지 않지만, 그 시점 이후로는 조건을 만족하는 경우인 최소값 찾기

매개 변수 탐색의 동작 과정
1. 문제에서 최종적으로 찾고자하는 답(최소값/최대값)을 매개변수로 취급한다.
2. 이 값을 매개 변수로 하는 결정 함수를 만들고 배열이 연속인지 확인한다.(아마 모든 경우를 넣어서?)
3. 답이 최소값이면 결정 함수 결과 배열에서 [false, false, ... true, true]에서 처음으로 true가 나오는 결과를, 최대값이면 마지막으로 true가 반환되는 지점이 답이 된다.
(배열값이 false, true가 뜻하는 바는 그 자체값을 뜻할 수도 있지만 문제에서 요구하는 조건에 대해 true, false를 뜻하는 것일 수도 있다. 즉 실제 함수값은 숫자일 수 있다.)

매개 변수 구간 정의
-> 이진 탐색과 마찬가지로 시작점, 끝점, 중간점을 변화시키면서 탐색한다. 
start, end, mid 변수가 있을때 mid 값일때 문제가 요구하는 조건에 true이냐 false이냐를 판단하여 이 mid값을 다음 반복문에서 start 또는 end 값으로 설정할 수 있다.
이때 가장 처음 start, end 값은 최소값/최대값, 문제가 요구하는 답의 종류가 무엇이냐에 따라 무조건 true,false(최대값) or false,true(최소값) 값을 정하게 된다.
(결정 함수 결과 배열에서 [f,f,f....,t,t,t] 등에서 양끝을 뜻한다. 즉 처음 start, end는 조건을 만족하는 가장 끝값(최소값이나 최대값)으로 설정한다.)
